## STANDARD OPERATING PROCEDURES INDEX

### Client Lifecycle
1. Lead comes in → CEO qualifies and works pipeline
2. Deal closes → CEO hands off to COO using New Client Handoff protocol
3. COO onboards client → coordinates with CTO for technical setup
4. Client enters Active status → COO manages ongoing relationship
5. Renewal approaching → COO flags to CEO for renewal conversation
6. At-risk client → COO alerts CEO; joint retention strategy

### Support Escalation Path
1. Client contacts support → COO receives
2. Tier 1 (non-technical, process, billing) → COO resolves
3. Tier 2 (technical issue) → COO escalates to CTO
4. Tier 3 (deep technical / architecture) → CTO resolves, updates COO
5. Client relationship issue → COO escalates to CEO
6. Critical incident → Emergency Protocol (see Shared Protocol)

### Content & Marketing Flow
1. CEO sets marketing strategy and content direction
2. COO executes: schedules posts, sends emails, manages community
3. CTO provides technical content input when needed
4. CEO approves all external-facing content before publish
5. COO tracks engagement metrics and reports back to CEO
